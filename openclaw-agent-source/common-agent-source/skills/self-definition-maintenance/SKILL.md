# Custom Identity and Rules Maintenance

## When to Use

- The user asks you to adjust your role positioning, boundaries, expression style, or delivery standards.
- The user asks you to improve identity/workflow documentation in the current workspace.

## Core Principles

1. Modify only the **current agent** workspace; do not modify other agents.
2. In `AGENTS.md`, only `AGENTS_USER_CUSTOM_SECTION` is editable.
3. Documentation updates must be actionable and verifiable; avoid vague wording.

## Execution Steps

1. Rewrite target docs based on user requirements:
   - `AGENTS.md`: operating rules and memory strategy (only `AGENTS_USER_CUSTOM_SECTION` can be edited)
   - `SOUL.md`: personality style, values, and behavioral boundaries
   - `USER.md`: owner profile, communication preferences, and delivery preferences
   - `IDENTITY.md`: your name, role positioning, temperament, and boundaries
   - `TOOLS.md`: local tool conventions, path rules, and usage boundaries
   - `HEARTBEAT.md`: short periodic checklist; keep concise
   - `MEMORY.md`: long-term high-value memory (avoid verbose logs)
2. Self-check:
   - Confirm edits are saved.
   - If `AGENTS.md` changed, confirm shared-rules area was not modified.
   - Explain behavioral changes after the edits.

## Output Requirements

- List modified file paths.
- Explain purpose and expected behavior change for each file.
- If anything is unfinished, provide next-step suggestions.
